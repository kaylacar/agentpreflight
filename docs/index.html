agentpreflight
GitHub npm

Pre-flight validation
for AI tool calls

Intercepts every file read, write, bash command, and git operation. Catches mistakes before they execute — not after.

npm install agentpreflight README →

What it catches

filesystem
  • File doesn't exist before read
  • Parent directory missing before write
  • No write permission
  • Sensitive file write (.env, .pem)
git
  • Commit with nothing staged
  • Force push to main/master
  • Diverged from upstream
  • --no-verify bypassing hooks
secrets
  • API keys written to files
  • Tokens in bash commands
  • OpenAI, Anthropic, GitHub, AWS, Stripe patterns
environment
  • OneDrive path redirect (Windows)
  • Wrong slash direction for OS
  • Unexpanded ~ paths
  • NUL vs /dev/null mismatch
scope
  • Path traversal out of working dir
  • Writes to /etc/, C:\Windows\
network
  • Dangerous protocols (file://, data://)
  • Internal network access
  • Secrets in request headers
  • HTTP instead of HTTPS
naming
  • Naming convention mismatch
  • Double extensions (.ts.ts)
  • Spaces in filenames
parallel
  • Cross-agent file conflicts
  • Concurrent git operations

Real examples

Blocked — unstaged commit
[FAIL] staging-verification: Nothing is staged for commit
       → Use git add to stage files first
Blocked — file doesn't exist
[FAIL] file-exists-for-read: File does not exist: ./config/settings.json
Blocked — hardcoded secret
[FAIL] secrets-in-file-content: Possible Anthropic API key detected in file content
       → Store secrets in environment variables, not in files

Claude Code global hook

Install once. Validates every tool call Claude makes, across every project.

1 — hook directory

mkdir -p ~/.claude/hooks && cd ~/.claude/hooks
pnpm init -y && pnpm add agentpreflight

2 — register in ~/.claude/settings.json

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Read|Write|Edit|Bash|Glob",
      "hooks": [{
        "type": "command",
        "command": "node C:/Users/YOU/.claude/hooks/preflight.mjs"
      }]
    }]
  }
}

3 — hook script + stats tracker

Full setup guide in the README. Includes a stats script that tracks total calls intercepted, blocks, and estimated tokens saved.

Use in a pipeline

import { createPreflight, hasFailures, formatResults } from 'agentpreflight';

const pf = createPreflight({
  rules: ['filesystem', 'git', 'secrets'],
});

const results = await pf.validate({
  tool: 'write_file',
  params: { path: './src/config.ts', content: fileContent },
});

if (hasFailures(results)) {
  console.error(formatResults(results));
  // don't execute the tool call
}
